## Animation Specifications

### Modal Entrance
```css
@keyframes fadeIn {
  from { opacity: 0; }
  to { opacity: 1; }
}

@keyframes slideUp {
  from {
    opacity: 0;
    transform: translateY(20px);
  }
  to {
    opacity: 1;
    transform: translateY(0);
  }
}

.modal-backdrop {
  animation: fadeIn 300ms ease-out;
}

.modal-content {
  animation: slideUp 300ms ease-out;
}
```

### Button Hover
```css
.primary-button {
  transition: all 300ms ease;
}

.primary-button:hover {
  transform: translateY(-2px);
  box-shadow: 0 8px 16px rgba(0, 165, 168, 0.3);
}
```

@keyframes successPulse {

.success-icon {

## Accessibility Notes

1. **Keyboard Navigation**
   - All buttons and links must be keyboard accessible
   - Tab order should be logical (top to bottom, left to right)
   - Modal should trap focus while open
   - ESC key should close modals

2. **Screen Reader Support**
   - All icons should have aria-labels
   - Status messages should use aria-live regions
   - Form fields must have associated labels

3. **Color Contrast**
   - All text must meet WCAG AA standards (4.5:1 ratio)
   - Interactive elements must be distinguishable
   - Don't rely on color alone for information

4. **Focus Indicators**
   - Visible focus indicators on all interactive elements
   - Use consistent focus style throughout
